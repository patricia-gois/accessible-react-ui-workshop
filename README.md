# ♿ Accessible React UI Workshop

A deep dive into building inclusive web interfaces using React, Vite, and WCAG 2.1 standards.

## 🌟 Overview
This project is a showcase of my journey in **Accessible Design & Frontend Development**. I've built three core exercises focused on overcoming common barriers for users with visual or motor impairments.

## 🚀 Key Features

### 01. Semantic & Accessible Navigation
- **Keyboard First:** Fully navigable via `Tab` and `Esc` keys.
- **Skip Links:** Implementation of "Skip to Main Content" for power users.
- **Mobile UX:** Accessible hamburger menu with `aria-expanded` and `aria-controls` states.

### 02. Empathetic Form Validation
- **Real-time Feedback:** Uses `aria-live="polite"` to announce errors without interrupting the user.
- **Input Association:** Every error is linked to its input via `aria-describedby`.
- **Focus Management:** Automatically moves focus to the first invalid field upon submission.

### 03. Theme-Aware Design System
- **Contrast Compliance:** Light and Dark modes designed to meet WCAG AA standards.
- **System Preference:** Automatically detects `prefers-color-scheme` via React Context.
- **CSS Variables:** Built with design tokens for a scalable and consistent UI.

## 🛠️ Tech Stack
- **React 18** (Hooks, Context API)
- **Vite** (Build Tool)
- **CSS3** (Custom Properties & Flexbox)
- **Lucide React** (Icons)
